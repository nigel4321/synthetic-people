# Most bin/ scripts run on Python 3.8+ stdlib only (argparse, glob,
# hashlib, json, os, re, subprocess, collections). Two stages need
# Python packages on the executor host:
#
#   - MULTIQC needs `multiqc` importable as a module
#     (`python3 -m multiqc`).
#   - PCA_PLOT (bin/plot_pca.py) needs numpy + sklearn + matplotlib
#     for the per-VCF cohort PCA scatter. Missing any of those is not
#     fatal — the script writes a tombstone JSON and a zero-byte PNG
#     so the pipeline doesn't abort, but no PCA is produced.
multiqc>=1.18
numpy>=1.24
scikit-learn>=1.3
matplotlib>=3.7
# rich 15.0 dropped the implicit ``rich.panel`` submodule re-export from
# ``rich/__init__.py``. multiqc 1.34 (current latest) still accesses
# ``rich.panel.Panel(...)`` directly inside its exception-rendering path,
# so a primary MultiQC error (e.g. "No datasets to plot" on tiny test
# fixtures) cascades into ``AttributeError: module 'rich' has no
# attribute 'panel'`` and obscures the diagnosis. Pin to <15 until
# multiqc upgrades its rich usage.
rich<15
